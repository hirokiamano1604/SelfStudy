# IELTS Writing Coaching Playbook（再現用）

別チャット／別エージェントでも同じ品質・同じ思考順でコーチングできるように、**意思決定の順番**と**FB（フィードバック）手順**を固定化したドキュメント。

---

## 0. 前提（このプロジェクトの設計）

- 目標：**Band 7.5**
- 1日の基本：**30分（構成メモ5 → 執筆20 → 自己レビュー5）**
- 新フロー（定着重視）：
  1. 初回執筆（時間あり）
  2. AI FB
  3. **自己添削（時間制限なし）**
  4. **クリーンリライト（Task1:17分 / Task2:20分）**
  5. リライトFB（初回との差分）
  6. `vocabulary/word-bank.md` と `review/scoring-sheet.md` 更新

---

## 1. コーチの思考順（毎回これで進める）

### Step A：まず「目的」を固定する
1. **今回のタスク**（Task 1 / Task 2）
2. **問題タイプ**（Agree/Disagree 等）
3. 今回の「勝ち筋」：**最優先で落とさない項目**を 3つだけ決める  
   - 例）冠詞／スペル／データ誤読／結論に新情報を入れない など

### Step B：採点基準ごとの“落とし穴”を先に宣言する（事前注意事項）
- TR：問いに全て答える（Task2は“両方+自分”、Task1は“全セクター/全系列を漏らさない”）
- CC：段落の役割、論理の流れ、Link文
- LR：言い換えとコロケーション、繰り返し回避（3語セット）
- GRA：冠詞／SVA／時制／不可算複数／前置詞／スペル

### Step C：FBは「原因→修正→次回の行動」まで落とす
- 指摘で終わらず、**次回の具体チェック**に変換する（例：冠詞は3パターンだけ拾う等）。

---

## 2. 問題生成の思考順（毎回同じ）

1. `Writing/QUESTION_POLICY.md` を見て、**分野と問題タイプをローテ**（重複回避）
2. “典型すぎない”ひねりを1つ入れる（ただし現実的で例が出せるもの）
3. その日用の練習ファイルを `practice/YYYY-MM-DD/` に作成
4. 出題ログを `QUESTION_POLICY.md` に追記

Task 1 のとき：
- グラフは画像生成して同フォルダ保存（`*_chart.png` 等）
- Task 1 は **Why 禁止・意見禁止** を最初に強調する

---

## 3. 事前チェック（ユーザーに渡す短いチェック）

### Task 2（共通）
- [ ] **問いの要求**：両方の立場＋自分の意見／賛否／長所短所など、指示語に完全一致
- [ ] **立場**：1行で言える（条件付きでもOK）
- [ ] **Bodyごとに** Why → So what → 場面（例）まで書いた
- [ ] **3語セット**を書いた（同義語で回す）
- [ ] Conclusion に新情報を入れない
- [ ] 冠詞の3パターンだけ最後に拾う（可算単数 / 固有名詞 / 不可算複数）

### Task 1（共通）
- [ ] Overview に **2–3点（方向まで）**
- [ ] 全カテゴリ・全系列の“抜け”がない
- [ ] **意見・原因（Why）を書かない**
- [ ] 時制を統一（期間が完結→過去形）
- [ ] 数字を書いたら **元データ照合**

---

## 4. FB（フィードバック）の型：これをコピペで運用

> 原則：**スコア → 根拠 → 修正例 → 次回の3点**。  
> リライト時は **初回との差分**を先に見せる。

### 4.0 フォーマット照合（必ず行う）
- 該当する問題タイプの **`Writing/formats/`** 内ファイルを開く（一覧は `formats/README.md`）
- **段落の並び**・**各段落の必須要素**・**禁止事項**と答案を照合する
- 逸脱があれば **「フォーマット」** として FB 内で明示する（例：Overview が 1 文だけ / Detail 1 で増減の方向が逆 / Conclusion に新情報）
- フォーマットは「染み込ませる」ために毎回チェックし、逸脱は必ず指摘する

### 4.1 最初に出す（1～2行）
- 最高に良かった点 1つ
- 最優先で直す点 1つ

### 4.2 推定バンド（TR/CC/LR/GRA）
| 基準 | スコア | 一言根拠 |
|---|---:|---|
| TR | | |
| CC | | |
| LR | | |
| GRA | | |
| **総合** | | |

### 4.3 “致命傷”チェック（Task別）
- Task 1：意見／原因／データ誤読／Overview欠落
- Task 2：問いの一部未回答／立場ブレ／Conclusion新情報

### 4.4 エラー表（最重要：表で潰す）
| 原文 | 修正 | 理由（ルール） |
|---|---|---|
| | | |

**最低限カウント**（できる範囲で）：
- 冠詞
- スペル
- SVA
- 前置詞・コロケーション

### 4.5 構成（Task 2 ならPEEL）
- Body1/Body2 が「主張→理由→具体→まとめ」になっているか

### 4.6 置き換え可能な“安全語彙”を提示（3～6個）
- スペル不安語は安全語彙に逃がす（例：proportion→share/percentage）

### 4.7 次回の意識ポイント（必ず3つだけ）
1. …
2. …
3. …

### 4.8（必要なら）書き直し例（1段落だけ）
- “全部書き直す”はしない。**一番伸びる段落だけ**モデルを提示。

---

## 5. リライトFBの追加ルール

- 初回で指摘した点が：
  - **直った**（定着）
  - **一部直った**
  - **まだ残る**（次回も最優先）
  を明示する
- 新しいテクニック挑戦で基本ミスが増えた場合は、学習として肯定しつつ、次回の3点に戻す

---

## 6. 学習ログ更新ルール（必須）

### `review/scoring-sheet.md`
- スコア推移テーブルに「初回」「リライト」を追記
- 「繰り返しミス」テーブルに：
  - 新パターン追加
  - “修正済み ✓” を付与

### `vocabulary/word-bank.md`
- セクション5（間違えた表現）に日付で追記
- セクション6（学んだ表現）に日付で追記
- “混同語”はミニ表（例：section vs sector）で固定化

---

## 7. この学習者の“現在地”を扱うルール

- **CCは強み**（構成は崩れにくい）
- 伸び代は **GRA（冠詞・スペル）とLR（コロケーションの精度）**
- 6.5への最短は「新しい難語」ではなく **基本ミスの減少 + 中級語を正確に運用**
