## 3. 単純ミスを減らす（土台）

### 冠詞（毎回チェック）
- [ ] **可算名詞の単数**に a/an がついているか（a huge proportion, a 5% share）
- [ ] **固有名詞**に不要な the をつけていないか（Country A, Japan → the 不要）
- [ ] **不可算名詞**を複数形にしていないか（consumption, information, well-being → s なし）
- 判定の順：数えられる？ → 単数？ → 特定？ → a/an/the を決める

### 複数形・SVA
- [ ] 主語が複数なら動詞は複数形（The pie charts **illustrate** / The figures **were**）
- [ ] 動名詞主語は単数扱い（Taking a gap year **provides** …）
- [ ] 不可算名詞は動詞は単数（The share **was** …）

### スペル（見直し1分で必ず通す）
- 危ない語：**their**（the+ir）, **interested**（interest+ed）, **beneficial**（benefit→beneficial）, **because**, **share**
- **自信がない語は安全語彙に逃がす**（proportion → share / percentage）

### 前置詞（コロケーションを崩さない）
- use X **for** Y（into ✗） / allocate X **to** Y
- the share **of** X / the figure **for** 2020 / **from** 10% **to** 50%
- care **about** / relate **to** / contribute **to**
- 同じ前置詞の繰り返しはある程度までOK。**正しさ**を優先する。

## 4. 繰り返しミス・癖（重点対策）

`review/scoring-sheet.md` の「繰り返しミスのトラッキング」と連動。**3回出たら重点対策**。

| 種類 | 対策 |
|------|------|
| **冠詞の脱落** | 「ひとつ」なら a/the のどちらか必須。可算単数・固有名詞・不可算複数の3パターンで拾う |
| section vs sector | 産業・区分は **sector**。文書の区画が section。 |
| a half → half | more than **half** of（a 不要） |
| resource vs source | ニュースの出所＝**source**。資源＝resource。 |
| where（場所以外） | 変化・結果のつなぎは and / with / 別文。where は場所だけ。 |
| the one of X（複数） | 複数なら **those of** X。単数なら that of X。 |
| Roughly（文頭） | 「全体として」は **Overall**。Roughly は「約〜」。 |
| leads to + 動詞 | leads to **動名詞**（maintaining 等）。to は前置詞。 |
| it's / its | its＝所有格。it's＝it is。his にアポストロフィをつけないのと同じ。 |

**定着済みの例**（維持する）：Conclusion に新情報を入れない、Task 1 で意見を書かない、use X for Y、the+固有名詞の排除、that of X、on balance。

## 5. Task 1 で必ず守ること

| やること | やらないこと |
|----------|--------------|
| **増減の方向をデータと必ず照合する** | 増えたのに decreased と書く |
| **数字を書いたら元の表・グラフと突き合わせる** | 写し間違い・方向逆 |
| **Overview を 2〜3 文で書く**（程度まで：sharply / slightly） | Overview を1文ですませる / 書かない |
| **全カテゴリ・全系列に触れる** | 5つあるのに3つしか書かない |
| **What だけ書く** | 意見・理由（Why）を書く（This is because... / I think... / tradeoff） |

- **書いてよい**: 数値、傾向、比較、Notably / It is worth noting that ...
- **書かない**: This is because ... / I think ... / It is interesting that ... / 推測・原因の説明
- **フォーマット**: Introduction → Overview → Detail 1（増えた）→ Detail 2（減った）。Detail の文型は型2・型4などバリエーションを決めてから書く。
- **時制**: 過去のデータ → 過去形で統一。語数は 150〜180 語目安。

## 6. Task 2 で必ず守ること

- **問いの全パートに答える**（both views + own opinion 等）
- **Conclusion に新情報を入れない**。要約＋立場の再確認のみ
- **立場を1文で明確に**。Introduction で分かるようにする
- **構成メモは日本語でロジックだけ**。Why → So what → 場面。英文は本文で書く
- **3語セット**を毎回書く（繰り返し回避）
- **Intro の具体語**: 各 view の「論点の定義」として 1語ずつ（autonomy, income volatility 等）。国名・企業名などの例は Body で

## 7. 構成・論理の習慣

### アイデア出し（Task 2）
- **軸で俯瞰する**: 心・体・時間・組織・個人vs会社。利点・欠点をそれぞれ1行ずつメモ。本番で使うのは2軸で十分。
- **3マス**: [施策・トピック] → [理由] → [ゴール]。メモの段階で「なぜ？」を日本語で1文書く。本文ではその「なぜ？」を必ず1文で書く。

### 段落の流れ（Task 2）
1. トピックセンテンス — 何について話すか  
2. 具体的な説明 — なぜそう言えるか（メカニズム・一般論）  
3. 解釈 — So what の手前  
4. 結果 — だから何が起きるか  
5. 具体例（あれば）— 1つの場面で示す  

- **具体説明** = メカニズム・一般論（やや抽象〜中程度でOK）  
- **具体例** = 想像できる1シーン（調査や固有名詞は不要）。目的は説得より**説明をわかりやすくする**こと。

### 構成メモ
- メモでは **↑ 増えた / ↓ 減った** を先に決め、**表・グラフと照合**してから本文を書く。メモの段階で方向を間違えるとそのまま本文に乗る。
- Topic sentence：Detail の1文目は「何について話すか」で十分。数値は2文目以降でよい。

## 8. 反省・教訓（練習ログから）

| 教訓 | 出典・補足 |
|------|------------|
| **議論の「フェアさ」** | 極端な主張（only/just）を避け、譲歩（Although/Even though）を入れると TR が伸びる |
| **自信がない語は安全語彙に逃がす** | スペルミスが多発した日の対策。proportion → share/percentage など |
| **SVA は意識すればゼロにできる** | 3日連続でゼロになった実績あり。見直しで必ず確認 |
| **Task 1 の致命傷は「方向逆」と「意見混入」** | 1つでもあると 6.5 に届かない。数字を書いたら必ず照合 |
| **リライトで定着を検証する** | 何も見ずに書き直すと、添削で学んだことが身についているか分かる |
| **新しいテクニック挑戦で基本ミスが増えても、学習として肯定** | ただし次回の3点（意識ポイント）には基本ミス対策を戻す |
| **6.5 は「難しい表現を増やす」より、致命傷ゼロ＋単純ミス削減＋Overview で程度まで** | Task 1 で 6.5 を狙うときの優先順位 |
