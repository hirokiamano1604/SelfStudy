# Task 2 アイデア出し・論理の Tips

議論の軸の出し方と、論理のつなぎ方・具体性のバランスをまとめたメモ。構成前にここを開いて確認する。

---

## 1. アイデア出し：軸で俯瞰する

**パッと思いついた1つが「どの軸」で見ているかを意識すると、他の軸が思い浮かびやすくなる。**

### よく使える軸（利点・欠点の両方に使う）

| 軸 | 利点の例 | 欠点の例 |
|---|---|---|
| **心** | リフレッシュ、集中の回復、ストレス減 | 集中が切れる、戻るまで時間がかかる |
| **体** | 血流、姿勢、目の疲れ、健康維持 | （話題による） |
| **時間** | 長期的な生産性アップ | 休憩分だけ労働時間が減る |
| **組織** | チームの連携、コミュニケーション | スケジュールがずれる、会議が割る |
| **個人 vs 会社** | 従業員の満足・健康 | 会社は稼働時間の減少と見る |

### 練習のやり方

- 毎回、**心・体・時間・組織**をメモに並べ、利点・欠点をそれぞれ1行ずつ書く。
- 本番で使うのはそのうち**2つで十分**。まず思いついた軸 + 別の軸を1つ足す。

---

## 2. 論理のつなぎ：型を固定する

### 理由は1論点あたり1本でよい

1本の理由を「具体説明 → 解釈 → 結果」まで書けば、論理としては足りる。

### 段落の流れ（5ステップ）

1. **トピックセンテンス** — 大まかに導入（何について話すか）
2. **具体的な説明** — なぜそう言えるか（メカニズム・一般論）
3. **解釈** — それはどういうことか（So what の手前）
4. **結果** — だから何が起きるか（結論・ゴール）
5. **具体例**（あれば） — その流れを1つの場面で示す

**書くときのコツ**: メモの段階で「**なぜ？**」を日本語で1文書く。本文ではその「なぜ？」を必ず1文で書く。つなぎは「So 〜」「This means that 〜」で後から足す。

### 3マスで整理（構成メモ用）

```
[施策・トピック]  例：短い休憩を取る
       ↓ なぜそう言える？
[理由]  例：ずっと座っていると集中が落ちる / 血流が悪くなる
       ↓ だから何？
[ゴール]  例：効率が上がる / 健康が維持される / ミスが減る
```

## 3. 具体説明 vs 具体例のレベル感

**役割が違うので、具体性の度合いを揃えなくてよい。**

| | 役割 | 具体性の目安 |
|---|---|---|
| **具体説明** | 「なぜそうなるか」の**メカニズム・一般論** | やや抽象〜中程度でOK。「長時間同じ作業をすると集中が落ちる」など、誰にでも当てはまる理屈。 |
| **具体例** | その理屈を**1つの場面で見せる** | 「例えば、オフィスで5分歩いて戻った人が、その後のミスが減った」など、誰か・どこか・何をしたかが分かる程度。 |

### 調整のコツ

- **具体説明がすでに詳しい**  
  → 具体例は短く「例えば、数分歩いて戻ると集中しやすくなる」程度でよい。

- **具体説明が抽象的**（「休憩するとリフレッシュできる」だけ）  
  → 具体例で中身を足す。「例えば、短い散歩やストレッチの後は、同じ作業でもミスが減ったり判断が速くなったりする」のように、**何をした → 何が良くなった**を1文で。

- **具体例は「想像できる1シーン」で十分**  
  調査や固有名詞は不要。IELTS では「例えば、オフィスワーカーが〜」レベルでよい。目的は説得より**説明をわかりやすくする**こと。

---

## 4. Intro と Conclusion の役割の違い（Task 1 vs Task 2）

- **Task 1**: Conclusion がない → **Overview** で「全体の傾向・何が言えるか」をまとめる。早めに概要を出す。
- **Task 2**: Conclusion がある → **Introduction** は問題の言い換え＋「〇〇と△△を論じる」という**予告だけ**。具体的な中身（原因・解決策のキーワード）は書かない。**Conclusion** で Body のキーワードを使って初めて具体的にまとめる。

→ Intro を大まかにしておくと、Conclusion のまとめと被らず、役割がはっきりする。
