# Task 1 で気をつけること（傾向・フォーマット・文法）

今日のタスクに関係なく、Task 1 を書くたびにここを開いて確認する。

---

## 1. 致命傷を絶対に防ぐ（TR）

| やること | やらないこと |
|---|---|
| **増減の方向をデータと必ず照合する** | 増えたのに decreased と書く（あなたの過去の致命傷） |
| **数字を書いたら元の表・グラフと突き合わせる** | 45% を 40% と写すなどの誤読 |
| **Overview を 2〜3 文で書く** | Overview を1文ですませる / 書かない |
| **全カテゴリ・全系列に触れる** | 5つあるのに3つしか書かない |
| **What だけ書く** | 意見・理由（Why）を書く |

### 意見・理由の禁止（Task 1 の鉄則）

- **書いてよい**: 数値、傾向、比較（increased, remained stable, the largest）。
- **書かない**: This is because ... / I think ... / It is interesting that ... / tradeoff / 推測・原因の説明。
- **代わりに使う**: Notably, ... / It is worth noting that ... （「注目すべきは」程度の事実の強調ならOK）。

## 2. フォーマット（構成の型）

- **並び**: Introduction → **Overview** → Detail 1 → Detail 2。順序を変えない。
- **比較系**（円グラフ2つ・表・国別棒グラフなど）:  
  **Detail 1 ＝ 増えたもの**、**Detail 2 ＝ 減ったもの** に分ける。  
  こうすると「方向の書き間違い」が減る。
- **時系列系**（折れ線など）: Detail 1 で前半・増加した系列、Detail 2 で後半・減少した系列など、役割を分ける。
- 詳細は `formats/task1_graph_trend.md` と `formats/task1_graph_compare.md` を参照。

---

## 3. Overview の質（6.5 を狙うなら）

- 「〇〇が増えて、△△が減った」の **二分だけ** で終わらせない。
- **程度** まで書く: 大きく増えた（sharply / markedly）、少し増えた（slightly）、最も減った（the steepest decline）など。
- 可能なら **最大・最小** を1つずつ: the largest share / the highest figure / the lowest など。

### 「傾向の一文」とは（Overview の核）

**定義**: 「どのカテゴリが増えたか・減ったか・安定か」を、**数値を出さずに** 1 文で言い切る文。Overview の **第 1 文** にするとよい。

**中身**: 主語（カテゴリ名 or 数値の主体）＋ 動詞（increased / decreased / remained stable）＋ 必要なら期間（over the period）。

| パターン | 例文 |
|---|---|
| A と C 増・B 減 | **Overall, visitor numbers to Museum A and C increased over the period, while those to Museum B decreased.** |
| 言い換え | **Over the period, Museum A and C saw a rise in visitor numbers, whereas Museum B saw a fall.** |
| 3 系列が同じ動きなら | **Overall, all three categories increased between 2015 and 2020.** |
| 増・減・横ばいが混在 | **Overall, A and B rose, C fell, and D remained stable.** |

**これが「傾向の一文」**。この 1 文があれば「全体の傾向は書いた」と判断される。2 文目で程度（sharply / slightly）や順位を足してよい。

## 4. Detail の文型バリエーション（「単調」を避ける）

同じ「A は 2015 年 20、2020 年 35」でも、**文の型**を変えると単調にならない。以下は同じ内容の **5 パターン**。Detail では 1 系列に 1 型だけ使い、別の系列で別の型を使うとよい。

| 型 | 例文 |
|---|---|
| **型 1** 主語 + was + 数値 / reached + 数値 | In 2015, the figure for Museum A was 20 thousand; it reached 35 thousand in 2020. |
| **型 2** 主語 + rose/increased from X to Y | Museum A’s visitor numbers **rose from** 20 thousand in 2015 **to** 35 thousand in 2020. |
| **型 3** By + 年、主語 + had reached X, up from Y | **By 2020**, Museum A **had reached** 35 thousand visitors, **up from** 20 thousand in 2015. |
| **型 4** 主語 + saw + 名詞（an increase from X to Y） | Museum A **saw an increase from** 20 thousand in 2015 **to** 35 thousand in 2020. |
| **型 5** 数値が主語（X in 年 + 動詞 + Y in 年） | **Visitor numbers** for Museum A **stood at** 20 thousand in 2015 **and** 35 thousand in 2020. |

**使い分けの例**（Detail 1 で A と C を書く場合）:
- Museum A を **型 2** で: *Museum A’s visitor numbers rose from 20 thousand in 2015 to 35 thousand in 2020.*
- Museum C を **型 4** で: *Museum C saw an increase from 15 thousand in 2015 to 26 thousand in 2020.*

こうすると「was / reached / became」の繰り返しにならない。**書く前に「この段落では型 2 と型 4 を使う」と決めておく**と楽。

## 5. 文法・語彙であなたがやりがちなこと（傾向）

### 冠詞

- 可算名詞の単数に **a/an** を付ける（a huge proportion, a 5% share）。
- **固有名詞に the を付けない**（Country A, Japan → the 不要）。
- 不可算名詞を複数にしない（consumption, information）。

### SVA

- 主語が複数なら動詞は複数形: The pie charts **illustrate**（illustrates ✗）。
- The figure / The share は単数 → was / increased。

### 語の選び方・スペル

| よくあるミス | 正しい形 |
|---|---|
| **section**（産業・区分の意味で） | **sector**（産業区分は sector。section は文書の区画） |
| **resource**（ニュースの出所で） | **source**（情報源＝source。資源＝resource） |
| **a half** of | **half** of（a 不要） |
| **Roughly,** the ...（文頭で「全体として」） | **Overall,** the ...（Roughly は「約〜」） |
| use X **into** Y | use X **for** Y / allocate X **to** Y |
| **where** it decreased（変化のつなぎで） | **and** it decreased / **with** the figure falling（where は場所のみ） |
| **four** different sources（5つある場合） | **five**（カテゴリ数を数えてから書く） |

### スペル（見直し1分で確認）

- their（the + ir）, share, overall, percentage, addition, minutes, than（比較）

## 6. 構成メモの習慣

- メモでは **↑ 増えたもの / ↓ 減ったもの** を先に決め、**表・グラフと照合**してから本文を書く。
- メモの段階で方向を間違えると、そのまま本文に乗る。必ず表の数値と突き合わせる。

---

## 7. 時制・語数

- 過去のデータ（2010年、2020年など）→ 動詞は **過去形で統一**（was, increased, declined）。
- 語数は **150〜180語** を目安に。書きすぎると Task 2 の時間が削られる。

---

## 8. Task 1 で 6.5 を狙うときに気をつけること

6.0 は「問いに答えているが方向ミスや不足あり」。6.5 は「致命傷なし＋主要な特徴をきちんと伝えている＋文法ミスが少ない」。

### 優先順位

| 優先度 | やること | 効果 |
|---|---|---|
| **1. TR の致命傷をゼロに** | 増減の方向照合、数字照合、Overview 2〜3文、全カテゴリ、意見禁止 | 方向・数字を1つでも間違えると 6.5 に届かない |
| **2. GRA の単純ミスを減らす** | 冠詞3パターン、SVA、スペルを見直しで必ずチェック | ミスが重なると 5.5 で止まる。減らすだけで 6.0→6.5 になりやすい |
| **3. Overview を「程度」まで書く** | 大きく増えた（sharply）/ 少し増えた（slightly）/ 最も減った など | TR と CC の印象が上がる |
| **4. LR は「正確さ」優先** | 3語セットで繰り返しを避ける。sector/source、use for などコロケーションを崩さない | 難語より正確な語の方が 6.5 では効く |
| **5. フォーマットを守る** | Detail 1＝増えた、Detail 2＝減った。段落の並びを崩さない | CC はすでに強み。型を守れば維持できる |

### 6.5 の境目

- **TR**: 方向・数字の誤りが **0**。Overview で 2〜3 点、全カテゴリに触れている。
- **GRA**: 冠詞・SVA・スペルのミスが **合計で 3 個以下** 程度に収まっている。
- **LR**: 同じ語の繰り返しが目立たず、**コロケーションの誤りが 1〜2 個以下**。
- **CC**: 構成が明確で、Overview → Detail 1 → Detail 2 の流れがはっきりしている。

**まとめ**: 6.5 は「難しい表現を増やす」より、**致命傷をゼロにし、単純ミスを減らし、Overview で程度まで書く**の 3 つで届きやすい。
